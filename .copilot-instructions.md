# Tempos HR - Mapa de Contexto Maestro para Copilot

Este archivo es la guía de navegación para Copilot. Úsalo para entender dónde reside el conocimiento del proyecto y cómo interactuar con sus diferentes partes.

## 1. Guía de Navegación (¿Dónde mirar?)

### 🛠️ Desarrollo y Lógica
- `/Backend/src`: Núcleo de la API, entidades y lógica de negocio.
- `/Frontend/src`: Interfaz de usuario, componentes y estilos.
- `/openspec`: Especificaciones de API y contratos de datos.

### 📚 Documentación y Conocimiento
- `/Documentation`: Documentación técnica detallada.
- `/INDEX-S1.md`: Índice maestro del estado actual del proyecto (Sprint 1).
- `/AUDIT_GUIDE.md`: Guía de estándares de calidad y auditoría.
- `/VALIDATION-CHECKLIST.md`: Lista de comprobación para validar cambios antes de commit.

### 🤖 Agentes y Workflows
- `/.agents/AGENTS.md`: Define el comportamiento y personalidades de los agentes que trabajan en este código.
- `/.agents/workflows`: **CRÍTICO.** Contiene los flujos de trabajo paso a paso:
  - `dev.md`: Instrucciones para iniciar el entorno de desarrollo.
  - `build.md`: Cómo generar la versión de producción.
  - `frontend-hardening.md`: Reglas para fortalecer el frontend.

### 🚀 Despliegue y Entrega
- `/DEPLOYMENT-S1.md`: Guía paso a paso para el despliegue.
- `/RESUMEN-EJECUTIVO-S1.md`: Resumen de alto nivel para gerencia.
- `/PR-TEMPLATE-S1.md`: Plantilla para Pull Requests.

## 2. Reglas de Comportamiento para Copilot

### A. Consulta de Workflows
Antes de proponer un cambio estructural o de despliegue, Copilot debe consultar los archivos en `/.agents/workflows`.

### B. Respeto Multi-tenant
Cualquier sugerencia de código en el Backend debe verificar la presencia de `companyId` en las consultas.

### C. Estándar ESM
Recordar siempre: los imports en archivos `.ts/.js` deben incluir la extensión `.js` para ser compatibles con el entorno de ejecución Node.js (ej. `import { x } from './y.js'`).

### D. Prioridad de Documentación
Si hay una duda sobre la arquitectura, la fuente de verdad es `/Documentation` y el `/INDEX-S1.md`.

## 3. Comandos Rápidos de Referencia
- Diagnóstico de DB: `node Backend/scripts/diagnostico.js`
- Reset de DB (Limpieza): `node Backend/scripts/reset_total.js`
- Servidor Dev: Seguir el workflow en `/.agents/workflows/dev.md`
